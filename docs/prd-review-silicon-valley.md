## 1. 产品定位（一句话）

**AI自主拓客Sales Agent，帮中小企业用自然语言定义客户，Agent闭环完成发现→画像→触达→成交。**

（28字）

**定位检验：** 不是"找线索工具"，不是"邮件群发器"，是**AI销售员**——用户说一句话，Agent替你跑完全程。

## 2. 用户故事地图

### 2.1 Happy Path（核心流程）

```
[中小企业老板/销售经理]
  │
  ├─ 定义目标："帮我找深圳的SaaS公司CTO"
  │   └─ Agent理解意图 → 拆解条件 → 生成采集任务
  │
  ├─ Agent自动采集线索
  │   └─ 多源采集 → 清洗去重 → 评分排序 → 呈现Top100
  │
  ├─ Agent生成客户画像
  │   └─ 行业分析 → 决策权限 → 痛点推断 → 意向评分
  │
  ├─ Agent个性化触达
  │   └─ LLM生成邮件/消息 → 选择最佳时机 → 发送
  │
  ├─ Agent持续跟进
  │   └─ 未打开→换内容重发 / 已打开→推送案例 / 已回复→安排会议
  │
  └─ 成交归因
      └─ 哪封邮件 → 哪个话术 → ROI多少
```

### 2.2 Edge Cases（必须处理的异常路径）

| # | 场景 | 当前处理 | 建议 |
|---|------|---------|------|
| E1 | 数据源API限流/封禁 | 多号轮换 | MVP先单源+手动导入，不强依赖外部API |
| E2 | 小红书/脉脉反爬封号 | 模拟真人频率 | **MVP不做私域引流**，先做邮件+企微 |
| E3 | LLM生成内容不专业（AI幻觉） | Prompt约束+安全检查 | 加人工审核环节，至少MVP阶段 |
| E4 | 客户投诉骚扰 | 频率控制 | 内置退订机制+DNC名单 |
| E5 | 邮件到达率低 | SPF/DKIM+域名预热 | Resend/SendGrid专业邮件服务 |
| E6 | 用户不会写目标描述 | — | 提供模板+行业向导，降低输入门槛 |
| E7 | 线索数据质量差（无效邮箱/电话） | 数据清洗+验证 | 多源交叉验证，有效率达80%才入库 |
| E8 | 成交数据无法追踪 | — | MVP阶段做线索交付，不做成交归因 |

## 6. MVP定义（≤2周开发量）

### 6.1 MVP一句话

**用户用自然语言描述目标客户 → Agent自动采集+评分+生成画像报告+AI邮件触达 → Dashboard看效果。**

### 6.2 MVP功能清单

| # | 功能 | 验收标准 | 天数 |
|---|------|---------|------|
| 1 | 自然语言→采集条件解析 | 输入"找深圳SaaS公司CTO"→正确拆解为条件 | 1d |
| 2 | 企查查API单源采集 | 采集100+条线索，有效率≥80% | 2d |
| 3 | 数据清洗+去重+评分 | 去重率≥20%，评分A/B/C/D分级 | 1d |
| 4 | AI画像生成 | 每个线索生成角色特征+痛点+意向评分 | 2d |
| 5 | LLM邮件生成+发送 | 每封≥2个个性化元素，送达率≥90% | 2d |
| 6 | 基础Dashboard | 线索数/评分分布/触达数/打开率/回复数 | 2d |
| 7 | 用户认证 | 注册/登录/JWT | 1d |
| 8 | 基础部署 | Vercel+Railway可访问 | 1d |

**总计：12个工作日（≤2周）**

### 6.3 MVP明确不做

- ❌ 多数据源（只做企查查）
- ❌ 企业微信触达（只做邮件）
- ❌ 触达序列编排（只做单次触达）
- ❌ A/B测试
- ❌ 多租户
- ❌ 私域引流（小红书/脉脉）
- ❌ 成交归因

### 6.4 MVP验收红线

1. 输入行业+职位 → 采集≥100条线索
2. AI生成个性化邮件 → 送达率≥90%
3. Dashboard显示核心指标
4. API P95 < 500ms

## 7. 产品风险（用户不买账的3个原因+应对）

### 风险1：线索数据质量不够，用户觉得"还不如我自己找"

**概率：** 40% | **影响：** 致命

**根因：** 企查查API权限有限，联系方式完整度低；很多中小企业数据缺失严重。

**应对：**
- MVP允许用户导入自有线索（CSV），Agent做画像+触达
- 长期：多源聚合（企查查+天眼查+工商公示+LinkedIn）
- 评分模型透明化，用户看到每条线索为什么是A/B级

### 风险2：AI生成的邮件"像机器人"，打开率低

**概率：** 35% | **影响：** 高

**根因：** LLM生成的邮件同质化，容易被识别为营销邮件；中文B2B邮件打开率本身就低。

**应对：**
- Prompt engineering深度优化（行业黑话、地域特征、公司动态）
- 3级个性化策略（姓名/公司 → 行业痛点 → 个人动态）
- A/B测试不同风格（专业型/故事型/数据型）
- 人工审核+修改机制（MVP阶段）

### 风险3：中小企业不愿付费，觉得"我自己用企查查也能找"

**概率：** 30% | **影响：** 高

**根因：** $29/月对小微企业仍是一个门槛；他们可能没有专人跟进线索。

**应对：**
- Freemium模式：免费体验10条线索+5封邮件
- 价值量化：显示"节省了XX小时人工""多触达了XX个客户"
- 按效果付费：线索转化为付费，而非按月订阅
- 行业垂直化：先做一个行业做透（如SaaS），建立口碑

## 8. 北极星指标

### 核心指标

**北极星 = 月活跃用户完成的「有效触达数」**

> 有效触达 = 被打开或被回复的触达次数

### 为什么是这个指标？

| 候选指标 | 问题 |
|---------|------|
| 线索采集数 | ❌ 数量≠质量，采集10000条无效线索无意义 |
| 邮件发送数 | ❌ 发送≠到达，更≠有效 |
| 注册用户数 | ❌ 注册≠使用，虚荣指标 |
| **有效触达数** | ✅ 直接反映Agent的价值——帮用户联系到客户了 |
| 付费用户数 | ❌ 滞后指标，无法指导产品迭代 |

### 支撑指标体系

| 类别 | 指标 | 目标（MVP后1月） |
|------|------|----------------|
| **获客** | 新注册→首次采集 转化率 | ≥50% |
| **激活** | 首次触达完成率 | ≥30% |
| **留存** | W2留存率 | ≥40% |
| **核心** | 有效触达率（打开+回复） | ≥25% |
| **收入** | 免费转付费率 | ≥5% |
| **推荐** | NPS | ≥30 |

## 附：评审总结

### ✅ 做对了的

1. **Agent自主决策**是正确方向——不是辅助工具，是替代人工
2. **自然语言交互**大幅降低使用门槛
3. **中文原生+企微**是真实差异化
4. **$29-99价格带**有市场空白

### ⚠️ 需要调整的

1. **MVP范围太大**——私域引流（小红书/脉脉）不该在Phase 1，建议砍掉
2. **数据源是最大瓶颈**——企查查权限不足是硬伤，需要确定替代方案
3. **成交归因过早**——MVP做线索交付+触达即可，成交追踪留给Phase 3
4. **冷启动策略缺失**——没有说清楚第一批用户怎么来

### 🔑 关键决策建议

1. **MVP先做"线索报告交付"而非"自动触达"**——降低合规风险，先验证线索质量
2. **行业垂直切入**——先做SaaS/科技行业，不要试图覆盖所有行业
3. **Freemium + 按量付费**——比纯订阅更适合中小企业
4. **Peter团队做前端+用户验证，Daniel团队做Agent+数据**——分工明确
