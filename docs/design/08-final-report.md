## 一、执行摘要 (Executive Summary)

拓客Agent是一个**AI-Native B2B智能获客平台**，目标成为中文市场首个AI Agent自主拓客产品。

**核心定位**：用户用自然语言描述目标客户 → AI Agent自主完成「发现→筛选→画像→触达→分析→优化」全闭环。

**市场机会**：
- 海外竞品（Apollo $99/月、Clay $349/月、ZoomInfo $15K/年）验证了市场需求
- 国内竞品（探迹、EC SCRM）AI能力弱，真正AI-Native产品空白
- 目标切入：中国B2B企业（SaaS/科技/制造），透明定价、企微原生

**技术优势**：
- **OpenClaw + FastAPI 混合架构** — OpenClaw做AI决策编排，FastAPI做稳定业务层
- 5个专用Agent协作：采集→画像→内容→触达→分析
- 月LLM成本仅 **$8-100**（GPT-4o-mini为主力），单次拓客100人约$0.27

**关键数字**：
| 指标 | 值 |
|------|------|
| MVP开发周期 | 2周 |
| 完整产品周期 | 8周(4 Phase) |
| MVP月成本 | ~$200 |
| 成长期月成本 | ~$2,000 |
| 目标定价 | $29-99/用户/月 |
| 毛利率预期 | 95% |

## 二、最优技术路线推荐

### ⭐ 推荐方案：OpenClaw + FastAPI 混合架构

**为什么不是纯OpenClaw？** 缺少持久化状态层、REST API和任务队列。
**为什么不是纯LangGraph？** 过度设计，学习曲线陡，与OpenClaw功能重叠。

```
┌────────────────────────────┐
│     客户端 (Next.js 14)     │
└─────────────┬──────────────┘
              │ HTTPS
┌─────────────▼──────────────┐
│      FastAPI 业务层         │
│  REST API + Celery队列      │
│  认证/限流/状态管理          │
└─────────────┬──────────────┘
              │ HTTP
┌─────────────▼──────────────┐
│    OpenClaw Agent 层        │
│  5个专用Agent + Skills      │
│  模型路由 + Fallback        │
└─────────────┬──────────────┘
              │
┌─────────────▼──────────────┐
│    数据层                    │
│  PostgreSQL+pgvector         │
│  Redis + MinIO              │
└────────────────────────────┘
```

### 技术栈确认

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端** | Next.js 14 + TailwindCSS + Shadcn/UI | 生态最强，SSR+AI SDK |
| **后端** | FastAPI (Python 3.11+) | AI生态最佳，异步原生 |
| **Agent** | OpenClaw (5个专用Agent) | 原生多Agent + Skills复用 |
| **数据库** | PostgreSQL 16 + pgvector | 关系+向量一体化，免费 |
| **缓存/队列** | Redis + Celery | 异步任务+缓存 |
| **LLM** | GPT-4o-mini(主力) + GPT-4o(复杂) + DeepSeek V3(中文) | 成本与质量平衡 |
| **邮件** | Resend ($0，免费tier) | 开发者体验好 |
| **部署** | MVP: Vercel+Railway+Supabase (~$85/月) | 最快上线 |

### 5个Agent分工

| Agent | 模型 | 成本等级 | 职责 |
|-------|------|----------|------|
| **Orchestrator** | opus46/glm5 | 高 | 主控决策，任务分发 |
| **Collector** | zai47 | 低 | 数据采集，成本敏感 |
| **Profiler** | GPT-4o-mini | 低 | 客户画像生成 |
| **Reacher** | GPT-4o-mini | 低 | 个性化内容生成+触达 |
| **Analyzer** | glm5 | 中 | 效果分析，报告生成 |

| **管理后台** | Dashboard+权限+模板+数据导入导出 | 4天 |
| **总计P0** | | **22天** |

### 数据流

```
用户定义目标 ("帮我找SaaS公司CTO")
    → Orchestrator理解目标
    → Collector多源采集
    → ETL清洗去重入库(PG+pgvector)
    → Profiler生成画像+评分
    → Reacher生成个性化内容
    → 多渠道触达(邮件/企微)
    → Webhook回调(打开/点击/回复)
    → Analyzer效果分析+报告
    → 自动优化下轮策略
```

### Break-even分析

| 定价模式 | 单价 | Break-even用户数 | 时间 |
|----------|------|------------------|------|
| $29/月/用户 | $29 | 7个付费用户 | 1-2月 |
| $49/月/用户 | $49 | 5个付费用户 | 1月 |
| $99/月/用户 | $99 | 3个付费用户 | <1月 |

**结论**：3-7个付费用户即可Break-even，风险极低。

## 六、Top 5 风险 + 缓解策略

| # | 风险 | 概率 | 影响 | 缓解策略 |
|---|------|------|------|----------|
| 1 | **LLM成本失控** | 中 | 高 | 三层模型(规则→小模型→大模型)、缓存、限额监控 |
| 2 | **邮件到达率低** | 中 | 高 | 专业邮件服务(Resend)、域名预热、SPF/DKIM配置 |
| 3 | **数据合规风险** | 中 | 极高 | 仅用公开数据源API、同意管理、脱敏、个保法合规 |
| 4 | **竞品跟进快** | 高 | 中 | 快速迭代MVP、深度中文本地化、企微原生集成是护城河 |
| 5 | **AI幻觉/内容质量** | 中 | 高 | Prompt约束+内容安全检查+人工审核节点 |
