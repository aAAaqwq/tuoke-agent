# 拓客智能体 — 硅谷级开发任务拆解

> **文档版本：** v1.0
> **创建日期：** 2026-04-02
> **创建人：** 小pm (PM Agent)
> **基于文档：** PRD v1.0 + CEO评估报告 + 13轮调研 + 系统架构 + 功能设计
> **任务粒度：** 每个任务 ≤ 1天工作量，含明确验收标准

---

## 1. 项目评估摘要

| 维度 | 评估 |
|------|------|
| **难度等级** | 🔴 **高** — 多平台对抗 + 数据源受限 + 成交闭环模糊 |
| **预估总工时** | **148 人天**（≈ 7.5 月单人 / 3.8 月双人） |
| **风险等级** | 🔴 **高** — 数据源合规 + 平台封号 + 冷启动无数据 |
| **MVP 工时** | **42 人天**（≈ 8.5 周单人 / 4.5 周双人） |
| **关键约束** | 企查查/天眼查 API 权限、小红书/脉脉反爬、个保法合规 |
| **推荐团队** | 2 后端 + 1 前端 + 1 AI/Agent + 0.5 PM（共 4.5 人） |

### 难度分级

| 模块 | 难度 | 工时占比 | 关键瓶颈 |
|------|------|---------|---------|
| 线索发现 | 🟡 中 | 15% | 数据源 API 权限 + 覆盖面 |
| 企业画像 | 🟢 低 | 10% | LLM 调用即可 |
| 私域引流 | 🔴 极高 | 30% | 小红书/脉脉反爬，封号风险 |
| 多平台触达 | 🔴 高 | 20% | 每平台 API/限频/风控不同 |
| 客户维护 | 🔴 高 | 15% | 记忆系统 + 人性化判断 |
| 成交优化 | 🔴 极高 | 10% | 非纯技术问题，需销售方法论 |

```

  数据源确认(0.7) → 采集Adapter(1.2-1.4) → 清洗评分(1.5-1.6) → 画像Agent(1.9)

关键路径总工时：约 42 人天（单人执行）
非关键路径可并行，双人可缩短至约 78 人天总历时
```

- 前端骨架(0.10) ‖ 后端骨架(0.9) ‖ AI 验证(0.11)

| # | 难点 | 影响 | 现状 | 解决路径 |
|---|------|------|------|---------|
| 1 | **数据源 API 权限不足** | 🔴 致命 | 企查查权限不足，天眼查无权限 | 工商公示数据(免费)+用户自有CRM+LinkedIn(海外) |
| 2 | **小红书/脉脉反爬极严** | 🔴 致命 | 无私信API，批量操作几乎不可能 | Phase 1 先走邮件+企微；小红书仅评论→引导关注→私信（<10条/天） |
| 3 | **个保法合规** | 🔴 高 | 收集企业数据是否合规存疑 | 只采集公开工商数据+用户自有数据；敏感数据脱敏+加密；法务审查 |
| 4 | **LLM 成本控制** | 🟡 中 | GPT-4o 单次画像≈$0.05 | 三层模型：GPT-4o-mini(日常)/GPT-4o(画像)/DeepSeek(批量) |
| 5 | **冷启动无历史数据** | 🟡 中 | BANT 评分模型无法校准 | 行业公开数据初始化+人工标注 100 条+持续校准 |
| 6 | **邮件送达率** | 🟡 中 | 新域名发送易进垃圾箱 | Resend 专业服务+域名预热 2 周+SPF/DKIM |
| 7 | **AI 幻觉（画像不准）** | 🟡 中 | 画像可能包含错误推断 | 置信度标签+安全检查+人工抽检 10%+反馈循环 |
| 8 | **多平台 API 碎片化** | 🟡 中 | 每个平台接口/限频/风控不同 | Adapter 模式统一接口，新渠道可插拔接入 |
| 9 | **成交率不可控** | 🟡 中 | 不是纯技术问题 | MVP 先验证线索质量，不追求自动成交 |
| 10 | **浏览器自动化被检测** | 🟡 中 | Playwright/Puppeteer 特征明显 | 指纹浏览器+随机延迟+模拟真人行为 |

## 8. 验收标准总表

### Phase 0 验收

- [ ] 数据源能力矩阵输出，至少 1 个可用数据源
- [ ] 合规审查报告输出，红线明确
- [ ] MVP 范围文档签字确认
- [ ] 项目骨架可运行，健康检查通过

### Phase 1 验收

- [ ] 输入行业+地区 → 采集 100+ 企业线索
- [ ] 清洗去重有效率≥80%，去重率≥95%
- [ ] AI 生成企业画像，含角色/需求/意向/建议 4 个维度
- [ ] BANT 评分模型输出 A/B/C/D 分级
- [ ] 企业画像报告可导出 PDF

### Phase 2 验收

- [ ] 邮件送达率≥90%
- [ ] 每封邮件包含≥2个个性化元素
- [ ] 触达序列自动执行，条件分支正确
- [ ] A/B 测试可对比不同模板效果
- [ ] 单渠道频率≤2次/周，静默时段不发送

### Phase 3 验收

- [ ] 客户记忆系统：下次触达引用历史
- [ ] 转化漏斗：发送→打开→点击→回复→会议 全链路可追踪
- [ ] ROI 报告：成本/收入/ROI 可计算
- [ ] 多租户隔离：不同租户数据互不可见
- [ ] RBAC：4 种角色权限正确

### Phase 4 验收

- [ ] 小红书触达：不封号前提下≥30%成功率
- [ ] AI 自动调优：话术/频率/时机有可见改善
- [ ] 开放 API 文档可用，含认证示例
- [ ] CRM 双向同步：线索+触达记录

### Phase 5 验收

- [ ] 敏感数据加密存储（AES-256）
- [ ] 用户可导出/删除个人数据
- [ ] API P95 < 200ms
- [ ] 前端 LCP < 2s
- [ ] 1000 QPS 压测通过
- [ ] 无 OWASP 高危漏洞
