## 1. 项目本质

**拓客智能体 ≠ 找线索工具，而是一个完整的 AI 销售员。**

用户（中小企业老板）定义目标客户画像 → Agent 自主完成：
```
线索发现 → 企业画像 → 私域引流（小红书/公众号/飞书/脉脉）
  → 个性化触达 → 客户维护 → 交付给用户 → 提高成交率
```

**目标客户**：有产品/服务但缺乏客户或目标客户不清晰的中小企业老板。
**核心价值**：将找到的潜在客户交付给用户，尽最大可能提高成交率。

## 2. 落地难度评估：🔴 高

### 各环节难度拆解

| 环节 | 难度 | 核心挑战 | 技术路径清晰度 |
|------|------|---------|---------------|
| **线索发现** | 🟡 中 | 数据源受限（企查查权限不足，脉脉无API，天眼查无权限） | 需确定替代数据源 |
| **企业画像** | 🟢 低 | LLM + 公开数据即可 | 清晰 |
| **私域引流** | 🔴 高 | 小红书/脉脉反爬极严，封号风险大 | 不清晰 |
| **多平台触达** | 🔴 高 | 每个平台 API/限频/风控不同 | 部分清晰 |
| **个性化内容** | 🟡 中 | LLM 能做，"不像机器人"需持续调优 | 较清晰 |
| **客户维护** | 🔴 高 | 需记忆系统 + 人性化判断时机 | 不清晰 |
| **成交率优化** | 🔴 极高 | 不是纯技术问题，需销售方法论 + 数据校准 | 不清晰 |

### 与 MediaClaw 的关键区别

| 维度 | MediaClaw | 拓客智能体 |
|------|-----------|-----------|
| 交互对象 | 自己的账号，自己发内容 | 别人的平台，接触别人的用户 |
| 风控风险 | 低（发自己的内容） | 高（主动私信 = 封号风险） |
| 技术可控性 | 高（浏览器自动化） | 低（平台对抗+合规） |
| 效果可衡量 | 发布成功即完成 | 需要成单才算成功 |
| 冷启动 | 容易 | 极难（无历史成单数据） |

## 3. 风险矩阵

| 风险 | 严重度 | 概率 | 缓解方案 |
|------|--------|------|---------|
| 数据源受限 | 🔴 高 | 高 | 多源聚合 + 合法爬虫 + 用户自有数据导入 |
| 平台封号 | 🔴 高 | 中 | 模拟真人频率（<10条/天）+ 多号轮换 + A/B 测试 |
| 法律合规 | 🔴 高 | 中 | 数据脱敏 + 用户授权 + 法律审查（小law） |
| 冷启动无数据 | 🟡 中 | 高 | 用行业公开数据初始化评分模型 + 人工校准 |
| 客户信任度 | 🟡 中 | 中 | 透明展示AI辅助 + 人工最终确认 |
| ROI 不确定性 | 🔴 高 | 高 | MVP 先验证线索质量，再扩展触达 |

## 4. 执行路径分析

### 路径不清晰的地方（小code/Claude Code 执行前必须明确）

1. **数据源**：企查查 API 权限不足、天眼查无权限、脉脉无 API → 实际数据从哪来？
   - 可能方案：工商公示数据（免费但慢）+ 用户自有CRM数据 + LinkedIn（海外）
   
2. **私域引流技术路线**：
   - 小红书：无私信 API，只能通过评论→引导关注→私信，极度受限
   - 脉脉：反爬极严，需要验证码+登录态，批量操作几乎不可能
   - 公众号：需要已有流量池，不是引流工具
   - 飞书：可以，但需要企业认证 + 对方也在飞书
   - 建议：Phase 1 先走**邮件 + 企业微信**，合规且可控

3. **成交闭环设计**：
   - AI 找到线索 → 自动发消息？还是生成推荐报告交给人工？
   - 定价策略谁定？销售话术谁写？
   - 建议：MVP 做**线索发现+画像报告交付**，触达交给用户

4. **冷启动**：无历史成单数据，BANT 评分模型参数怎么校准？

- [ ] 企业数据采集 → 清洗 → 结构化存储

- [ ] 个性化内容生成（LLM + 模板）
- [ ] A/B 测试框架

### Phase 3 — 闭环优化（4-6 周）
- [ ] 私域引流（小红书/脉脉）— 需要先验证合规方案
- [ ] 客户维护系统（记忆+时机判断）
- [ ] 成交率追踪+优化
- [ ] 多租户/定价
