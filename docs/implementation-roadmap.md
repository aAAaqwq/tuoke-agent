## 6. 回滚方案

### 6.1 各 Phase 回滚策略

| Phase | 回滚触发条件 | 回滚方案 | 数据处理 |
|-------|------------|---------|---------|
| Phase 0 | 数据源不可用 | 改用用户自有数据 + 手工导入 | 无数据 |
| Phase 1 | 线索有效率<70% | 降级为手动录入+AI辅助分析 | 清洗失败数据 |
| Phase 2 | 邮件送达率<80% | 切换邮件服务商 / 降级为"只生成话术，手动发送" | 保留触达记录 |
| Phase 3 | 闭环无法跑通 | 降级为"线索报告"模式，不做自动触达 | 保留分析数据 |
| Phase 4 | 某渠道被封 | 关闭该渠道，不影响其他渠道 | 保留已有触达记录 |
| Phase 5 | 安全漏洞 | 暂停服务，修复后重新部署 | 不影响数据 |

### 6.2 核心降级方案

**最保守的 MVP（"线索报告"模式）：**
```
用户输入目标客户画像 → AI 采集+画像+评分 → 生成线索报告 PDF → 用户自行触达
```
- 工时：仅需 Phase 0 + Phase 1 ≈ 35 人天
- 风险极低，合规无问题
- 缺点：无自动化触达，用户需手动操作

**标准 MVP（邮件触达模式）：**
```
用户输入目标客户画像 → AI 采集+画像+评分 → 自动生成邮件 → 自动发送+追踪 → 效果报告
```
- 工时：Phase 0 + Phase 1 + Phase 2 ≈ 56 人天
- 风险可控，合规有保障
- 缺点：只有邮件触达

### 6.3 数据安全回滚

- 数据库每日备份（Supabase 自动）
- 代码 Git 版本管理，任何变更可回退
- 部署使用 Docker + 版本化镜像，可快速回滚到上一版本
- LLM API Key 可独立吊销，不影响其他服务
