# 企查查 vs 天眼查 深度对比分析

## 1. 数据覆盖对比

### 企查查
- **数据来源**: 国家企业信用信息公示系统（最权威）
- **覆盖范围**: 2.5亿+企业
- **更新频率**: T+1（工商数据每日更新）
- **数据维度**:
  - 工商信息（最全）
  - 司法风险（法院公告、被执行人）
  - 知识产权（专利、商标、软著）
  - 财务数据（上市公司财报）
  - 招投标信息
  - 新闻舆情

### 天眼查
- **数据来源**: 工商数据 + 多渠道聚合
- **覆盖范围**: 3亿+数据节点（企业+个人）
- **更新频率**: T+1（部分数据实时）
- **数据维度**:
  - 工商信息
  - 司法风险
  - 知识产权
  - 舆情监控（更强）
  - 企业关系图谱（更丰富）
  - 职业背景（个人简历）

**数据覆盖结论**:
- 企查查：工商数据最权威、最全
- 天眼查：数据维度更丰富，舆情监控更强

### 1. 探迹核心卖点：智能推荐算法

#### AI智能推荐引擎

**算法架构**:
```
用户画像 + 企业画像 → 相似度计算 → 优先级排序 → 个性化推荐
     ↓           ↓           ↓           ↓
  行业/规模   经营数据     协同过滤    动态调整
  需求特征   行为轨迹     深度学习    效果反馈
```

**核心算法模型**:

1. **协同过滤推荐**
   - 基于成功案例的相似性
   - "与您相似的客户选择了这些企业"
   - 冷启动问题：用内容推荐补充

2. **深度学习画像匹配**
   - 企业特征向量（200+维度）
   - 用户需求向量（实时更新）
   - 余弦相似度计算

3. **行为序列预测**
   - 分析销售人员的浏览、联系、成交行为
   - 预测下一个高意向线索
   - LSTM时序模型

**推荐效果**:
- 精准度：相比随机筛选提升3-5倍
- 转化率：平均提升25-40%
- 时间节省：销售人员筛选时间减少60%

```
决策人标签：
```

#### 3.1 数据清洗（Data Cleaning）

**清洗规则**：

```
1. 格式标准化
   - 公司名称：全角转半角、去除空格

4. 缺失值填充
   - 行业：根据经营范围推断
   - 规模：根据招聘数量估算
   - 地区：根据注册地址提取
```

```python
def find_fuzzy_duplicates(names, threshold=90):
    duplicates = []
    for i, name1 in enumerate(names):
```

### 4. AI应用详解

#### 4.1 推荐算法（Recommendation）

**推荐策略**：

**1. 协同过滤（Collaborative Filtering）**
```
用户相似度 = 余弦相似度(用户行为向量)

示例：
用户A浏览了 [公司1, 公司2, 公司3]
用户B浏览了 [公司1, 公司2, 公司4]

相似度 = cos([1,1,1,0], [1,1,0,1]) = 0.67

推荐：公司4给用户A
```

**2. 内容推荐（Content-Based）**
```
企业特征向量 = [行业, 规模, 地区, 融资, 招聘, ...]

用户A成交客户：[科技公司, 100-500人, 北京, B轮]
新线索匹配度 = 余弦相似度(企业特征, 成交客户特征)

推荐：高匹配度企业
```

**3. 深度学习推荐（Deep Learning）**
```python
# 双塔模型（Two-Tower）
import torch
import torch.nn as nn

class TwoTowerModel(nn.Module):
    def __init__(self, user_dim, item_dim, hidden_dim):
        super().__init__()
        
        # 用户塔
        self.user_tower = nn.Sequential(
            nn.Linear(user_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim)
        )
        
        # 企业塔
        self.item_tower = nn.Sequential(
            nn.Linear(item_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim)
        )
    
    def forward(self, user_features, item_features):
        user_embedding = self.user_tower(user_features)
        item_embedding = self.item_tower(item_features)
        
        # 点积计算相似度
        score = torch.sum(user_embedding * item_embedding, dim=1)
        return torch.sigmoid(score)
```

```python
        intent = self.classifier.predict(text)
        
        return intents, entities, intent
```

#### 4.3 知识图谱（Knowledge Graph）

**图谱构建**：

```
节点类型：
- 企业节点（Company）
- 人物节点（Person）
- 产品节点（Product）
- 事件节点（Event）

边类型：
- 股权关系（OWN）
- 任职关系（WORK_AT）
- 交易关系（TRADE_WITH）
- 竞争关系（COMPETE）
- 供应链关系（SUPPLY）

图谱示例：
[公司A] --股东--> [张三] --任职--> [公司B]
   ↓
[产品X] --竞争--> [产品Y] <--生产-- [公司C]
```

**图数据库查询**（Cypher示例）：
```cypher
// Neo4j查询：找公司A的所有关联企业（2层）
MATCH (c1:Company {name: '公司A'})-[r*1..2]-(c2:Company)
WHERE c1 <> c2
RETURN DISTINCT c2.name, type(r[-1]), r[-1].share_ratio
ORDER BY r[-1].share_ratio DESC
LIMIT 10;
```

**图谱应用**：
1. 股权穿透（找实际控制人）
2. 关联风险（风险传导路径）
3. 竞品发现（竞争关系网络）
4. 客户推荐（相似客户网络）
